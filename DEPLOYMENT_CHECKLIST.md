# ✅ Checklist для развертывания

Используйте эот checklist, чтобы ничего не забыть при развертывании приложения.

- [ ] Установил Node.js (v16+ или v18+)
- [ ] Установил npm или yarn
- [ ] Создал GitHub аккаунт (если нет)
- [ ] Создал Supabase аккаунт (если нет)
- [ ] Создал Vercel аккаунт (если нет)

## 📦 ЛОКАЛЬНАЯ УСТАНОВКА

### Шаг 1: Cloning/Creating Project
- [ ] Склонировал репозиторий **ИЛИ** создал новую папку проекта
- [ ] Открыл терминал в папке проекта
- [ ] Проверил наличие `package.json`
- [ ] Проверил наличие `next.config.js`

### Шаг 2: Установка зависимостей
- [ ] Выполнил `npm install`
- [ ] Дождался завершения (обычно 2-3 мин)
- [ ] Нет ошибок в консоли
- [ ] Папка `node_modules` создана

### Шаг 3: Конфигурация окружения
- [ ] Создал файл `.env.local` в корне проекта
- [ ] Заполнил все переменные окружения:
  - [ ] `NEXT_PUBLIC_SUPABASE_URL`
  - [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - [ ] `SUPABASE_SERVICE_ROLE_KEY`
  - [ ] `ADMIN_PASSWORD`
- [ ] Файл `.env.local` в `.gitignore` (должен быть изначально)

### Шаг 4: Локальный тест
- [ ] Выполнил `npm run dev`
- [ ] Приложение запустилось без ошибок
- [ ] Открыл http://localhost:3000 в браузере
- [ ] Видна главная страница с кнопками
- [ ] Нажал на "Подать заявку" - открылась форма
- [ ] Форма загружается корректно

## 🗄️ НАСТРОЙКА SUPABASE

### Шаг 1: Создание проекта
- [ ] Зашел на https://supabase.com
- [ ] Создал новый проект
- [ ] Назвал проект `tiger-king-old-recruitment`
- [ ] Установил сильный пароль БД
- [ ] Выбрал регион (ближайший к вам)
- [ ] Проект инициализировался (~5 мин)

### Шаг 2: Получение ключей
- [ ] Открыл Settings → API
- [ ] Скопировал `Project URL`
- [ ] Скопировал `anon public` ключ
- [ ] Скопировал `service_role secret` ключ
- [ ] Не делился ключами с кем-либо

### Шаг 3: Создание таблицы
- [ ] Открыл SQL Editor в Supabase
- [ ] Создал новый query
- [ ] Скопировал SQL из SETUP_SUPABASE.md
- [ ] Выполнил SQL (нажал Run)
- [ ] Таблица `player_applications` создана
- [ ] Нет ошибок при выполнении

### Шаг 4: Проверка RLS
- [ ] Открыл Authentication → Policies
- [ ] Видны 3 политики (INSERT, SELECT, DELETE)
- [ ] Политики состояние: ENABLED

### Шаг 5: Локальный тест отправки
- [ ] Остановил локальный сервер (Ctrl+C)
- [ ] Обновил `.env.local` с ключами Supabase
- [ ] Запустил сервер снова (`npm run dev`)
- [ ] Открыл форму
- [ ] Заполнил и отправил тестовую заявку
- [ ] После отправки видно сообщение об успехе
- [ ] Открыл админ-панель (`/admin`)
- [ ] Введил пароль и видю заявку в списке

### Шаг 3: Развертывание на Vercel
- [ ] Открыл https://vercel.com
- [ ] Нажал на "Sign In" или "Sign Up"

- [ ] Выбрал репозиторий `tiger-king-old-recruitment`
- [ ] Нажал "Import"

### Шаг 4: Конфигурация Vercel
- [ ] В окне "Configure Project":
  - [ ] Framework Preset выбран "Next.js"
  - [ ] Root Directory = `./` (пусто)
  - [ ] Нажал на "Environment Variables"
- [ ] Добавил переменные окружения:
  ```
  NEXT_PUBLIC_SUPABASE_URL = https://xxx.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJ...
  SUPABASE_SERVICE_ROLE_KEY = eyJ...
  ADMIN_PASSWORD = TigerKingOld2024
  ```
- [ ] Нажал "Add" для каждой переменной
- [ ] Проверил, что все 4 переменные добавлены
- [ ] Нажал "Deploy"

- [ ] Дождался "Building..."

## 🆘 ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

| Симптом | Решение |
|---------|---------|
| "Build failed" | Посмотреть логи Vercel → Deployments |
| "Cannot connect Supabase" | Проверить ключи в Vercel Environment Variables |
| "Table does not exist" | Заново запустить SQL скрипт в Supabase |
| "Form not submitting" | Открыть F12 → Console и посмотреть ошибку |
| "Blank admin panel" | Попробовать F5 (обновить) и нажать "Refresh" |

Полная справка: [FAQ.md](./FAQ.md)
